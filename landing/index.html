How It Works

Three steps to give your AI agent full browser control

01

Enable Chrome Debugging

Start Chrome with remote debugging enabled. One flag, permanent setup with a .desktop file.

google-chrome --remote-debugging-port=9222
02

Connect the MCP Server

Add chrome-agent-mcp to your Claude settings. Run via npx — no global install required.

npx chrome-agent-mcp
03

Watch Claude Browse

Your AI agent gains full browser control. Navigate, click, fill forms, inspect DevTools — all autonomously.

Built for Reliability

Resilient by Design

Every tool call goes through a layered fault-tolerance system so your agent never gets stuck

Primary

20s Timeout

A generous primary timeout handles slow pages and heavy DOM operations.

→
Retry

2× Parallel Retries

On timeout: two concurrent retries at 10s. Reads are parallel, writes sequential.

→
Fallback

New Group Fallback

If all retries fail, a fresh Chrome group is created and the agent continues seamlessly.

Quick Setup

Up and Running
in 2 Minutes

Four steps to give your Claude agent full browser control

1. Launch Chrome with remote debugging

google-chrome --remote-debugging-port=9222

2. Run the MCP server

npx chrome-agent-mcp

3. Add to Claude Desktop config

{
  "mcpServers": {
    "chrome-agent-mcp": {
      "command": "npx",
      "args": ["chrome-agent-mcp"],
      "env": { "CHROME_DEBUG_PORT": "9222" }
    }
  }
}

4. Install the extension (optional)

Open chrome://extensions → Enable Developer mode → Load unpacked → select the extension/ folder
